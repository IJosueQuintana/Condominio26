//////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////
EJEMPLO DE COMO SERIA EL FLUJO: 
////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////

Desde registrarDeudaController , genera una NuevaDeudaDTO que se la envia a Service así:

	NuevaDeudaDTO nuevaDeuda = new NuevaDeudaDTO()

A esa nuevaDeuda se le llena con todo lo del formulario y se envia a Service:

	deudaService.registrarNuevaDeuda(nuevaDeuda)

El service que debe hacer:

* Recibe el DTO, lo desenvuelve y Crea una Deuda.
* Aplica Toda la lógica necesaria para poder obtener la Deuda que va a la base de Datos
* Delega al model y a los patrones ciertas actividades (state y strategy)
* Llamada al DAO para guardar la deuda


El DAO que debe hacer:

* Recibe algo para insertarlo en la base de datos.

Desde consultarDeudas:

- El consultarDeudasController captura la cédula del cliente desde el textField

- El controlador le envía la cedula al service de cualquier manera ejemplo:
	- deudasService.consultarDeudas(txtCedula.getText()) 

	O
	- String cedula = txtCedula.getText())

	- deudasService.consultarDeudas(txtCedula.getText())


El service que hace:
* Recibe la cédula y llama al DAO
* El DAO le devuelve una lista de las deudas que cumplan las condiciones
* El service delega a DeudaFactory, a los patrones State y Strategy y al model la creación de las clases respectivas
* El service genera un DTO que le entrega al controlador

El controlador recibe las listas de DeudaConsultaDTO

Les muestra y asi
